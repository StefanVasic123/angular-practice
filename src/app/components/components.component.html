RENDERING

There are three ways to call component inside another component:

  • selector as HTML tag
  • selector as Class
  • selector as Attribute

Selector as HTML tag

First, to show html as string we create variable htmlString inside this component (component.components.ts) and called it here.
This is output and this is what call of selector as HTML implementation look like: {{htmlString}}
Below we rendered example component with default content 'example-works!'.
Declaration for rendering some component we can see at ts (typescript) file of component we want to render and there inside @Selector object by key 'selector'.

Selector as Class

When we want to call some components as class name like this: {{classTemplate}}, all we need to do is to change value of selector by adding dot as prefix: ".app-example-class"
And set class name as selector value, in this case: app-example-class.

Selector as Attribute

Procedure is the same as selector as class but instead of a dot as a prefix we put parentheses ([]) inside @Component selector key in ts file of the folder: {{attributeTemplate}}
Also, unlikely selector as class, when we use method selector as attribute, we put selector value like attribute in the element, this is what look like component below that render text 'example-atribute works!': {{htmlAttribute}}
